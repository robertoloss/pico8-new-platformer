🚨 PROBLEM IDENTIFICATION
The 1-Pixel Penetration Occurs Because:

1. Collision Detection Looks At Wrong Position:
• Player's right edge is at screen x = 71 (world x = 6.7)
• Collision check looks at x = 71.3 (flag(71.3, 65, 0))
• This checks world position (7.3, 1) which is EMPTY space

2. The Right Tile is Actually at:
• World position (8,0) to (15,7)
• Screen position (71.7,64) to (79.7,71)

3. Visual Result:
• Player's right edge (world 6.7) to (world 7.7) = 1 pixel penetration
• The tile starts at world position 7.7, but player extends to 7.7

4. Root Cause:
• Collision detection at (nx_tl + 7, ny_tl) = (71.3, 65)
• Should check at (nx_tl + 8, ny_tl) = (72.3, 65) to prevent penetration
• But this would cause the "getting stuck" problem you mentioned
💡 THE REAL ISSUE
The problem isn't in collision detection - it's in the boundary calculation:

When a collision IS detected, hit_right = ceil((mx+p.vx)/8) * 8 positions the map so the player's LEFT edge aligns with a tile boundary, but doesn't account for the player's 8-pixel width.

For proper collision prevention:
• hit_right should be ceil((mx+p.vx)/8) * 8 - 8
• This ensures the player's RIGHT edge doesn't penetrate

But first, collision detection needs to be fixed to actually detect the collision:
• Change inc_x = 7 to inc_x = 8 for right movement
• This makes collision detection check one pixel beyond the sprite boundary
• Then adjust boundary calculations accordingly
